let and const

发表于 2019-03-25 | 分类于 ES6 notes

const 会阻止对于变量绑定与变量自身值的修改。并不会阻止对变量成员的修改

暂时性死区

使用 let 或 const 声明的变量，在达到声明位置之前都是无法访问的，试图访问会导致一
个引用错误，即便所用的操作通常是安全的，例如使用 typeof 运算符

for循环还有一个特别之处，就是设置循环变量的那部分是一个父作用域，而循环体内部是一个单独的子作用域。
